## 3. AEO (Answer Engine Optimization) — 2026 best practices

**Goal:** be the source ChatGPT / Perplexity / Google AI Overviews / Claude cite when answering questions.

| Practice | Status | Notes |
|---|---|---|
| FAQPage schema with question-answer format | ✅ ALREADY DO | Every blog has FAQPage JSON-LD with 5 Q&A pairs |
| 40-60 word "Answer Blocks" at top of pages | 🟡 PARTIAL | We have TL;DR bullet lists but not always tight 40-60 word direct answers. **Test:** add a concise answer paragraph at the top of each page right after the H1 |
| Content structure by intent type (definitional / process / comparison / decision) | 🟡 PARTIAL | We mostly do process (booking walkthroughs) + comparison (bonds vs loans). Decision intent (e.g., "should I post bail with an ICE hold?") is underused |
| Citation quality + topical authority | ✅ ALREADY DO | CA Penal Code citations, county-specific bail schedules, licensed bondsman signature |
| Schema markup completeness | ✅ ALREADY DO | Article + FAQPage + LocalBusiness on every blog |
| Entity recognition (clear schema.org entities) | ✅ ALREADY DO | LocalBusiness with name/phone/address/license |
| Original data / statistics | 🟡 PARTIAL | We cite CA codes; we don't have our own data (e.g., "in Q1 2026 we posted X bonds in Hacienda Heights"). Adding even monthly aggregate stats would boost AI citation likelihood |

**Action:** Add a 40-60 word "Answer Block" pattern to the blog template — a concise direct answer right after H1, before the TL;DR. Audit existing blogs to insert these.

## 6. Claude prompt caching — token efficiency

**Status:** Not directly relevant today (API credits = 0, all generation happens in Claude Code sessions). Reference for when/if we resume API usage.

Key facts for future:
- Cache reads cost 0.1× base input rate (90% savings on repeat inputs)
- Cache writes cost 1.25× base for 5-min TTL; 2× for 1-hr TTL
- 2026 change: TTL dropped from 60min → 5min default → costs went up 30-60% for many workloads
- Batch API: 50% off when async (24hr return) — stacks with caching

**Action:** When API resumes, design all script prompts with explicit `cache_control` annotations + use Batch API for non-urgent jobs. Memory note for future-EROS.

1. **Two missing capabilities** worth phased adoption: (a) Answer Blocks (40-60 word direct answer at top of page), (b) "Local Landmarks" section on every microsite. Both are high-leverage, low-effort.

2. **Two memory corrections** needed: (a) "Edward Stern" → "Edward Sturm" in `feedback_compact_keywords.md`, (b) add new memory `feedback_aeo_geo_2026.md` capturing Answer Blocks + GEO citation strategy.

4. **One unmet opportunity:** video content. Sturm's "multi-channel coverage" thesis (page + short video for same keyword) — we have zero video presence. Future phase.

5. **Validation:** our current architecture (FAQPage schema, LocalBusiness JSON-LD, conversational queries, compact city-specific keywords, ≥70% pure bond content rule) is already well-aligned with 2026 AEO + GEO standards. The marginal improvements are the Answer Block pattern and original-statistics addition.
